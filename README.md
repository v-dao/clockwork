# Clockwork

clockwork 是一个仿真平台，仿真对象是地球和地球上的实体，例如飞机、坦克、舰船、空面弹、空空弹、鱼雷、水雷、箔条弹、角反射体等；仿真对象也包括可见光、红外光、电磁波、雷达、干扰、电子侦察等方面仿真。**完整产品愿景**下，仿真实体的控制逻辑可基于 **Lua** 或**可视化蓝图**编排。**Lua 脚本当前暂缓**（不实现宿主、不执行想定中 `entity_script` 的 `lua` 路径），详见 [doc/development-plan.md](doc/development-plan.md)。**可视化蓝图不随 Lua 一并暂缓**，按同一文档**阶段 7**与里程碑推进；想定中可声明 `blueprint`/`bp` 引用，具体运行时与编辑器闭环以计划为准。

# 仿真平台组成
1. 想定编辑
    想定编辑可以根据注册的模型，配置模型参数生成型号数据；
    想定编辑可以编辑仿真实体的控制逻辑
    想定编辑可以预先设定航线、空域信息
    想定编辑可以预先设置仿真通信网络
    想定编辑可以组装型号为仿真平台，可以设置仿真平台数据，名称、2维图标、3维模型、属性等；
    想定编辑可以部署仿真平台到场景中，作为仿真实体，部署位置、名称等；

2. 仿真引擎
    仿真引擎可以读取限定信息初始化仿真实体
    仿真引擎可以接收控制指令（初始化、开始、暂停、结束、设置仿真倍速）
    仿真引擎可以输出仿真态势信息
    仿真引擎拥有分布式仿真的能力，可以和其他电脑上的仿真引擎数据通信，让模型可以与分布式的模型数据交互
    仿真引擎拥有保存快照并且恢复快照的能力（仅本地单机仿真；分布式联邦模式下不提供、不允许使用快照）
    
3. 仿真模型
    仿真模型有mover类型，负责仿真实体运动的能力；
    仿真模型有sensor类型，负责仿真实体探测其他实体的能力；
    仿真模型有comdevice类型，负责仿真实体之间通信的能力；
    仿真模型有processor类型，负责处理实体自身的信息处理能力；
    仿真模型有weapon类型，负责仿真微波武器或者弹药发射架的能力；
    仿真模型有signature类型，负责描述仿真实体的可见光特性、红外特性、电磁特性等；

4. 态势显示（拥有二维地图与三维地球的功能）
    态势显示拥有显示二维地图的能力
    态势显示拥有显示三维地球的能力
    态势显示拥有显示想定中航线的能力
    态势显示拥有显示想定中空域信息的能力
    态势显示拥有显示仿真实体的能力
    态势显示拥有显示仿真实体坐标轴与实体矢量信息的能力（速度矢量，转动矢量）
    态势显示拥有显示雷达波束、干扰波束、雷达包络的能力

5. 仿真运行
    仿真运行程序用于组合仿真引擎和态势显示功能，把引擎中的仿真实体绘制到态势显示中
    仿真运行程序提供人机交互界面

# 开发
1. 使用C++开发
2. 使用C++20标准
3. 使用g++编译器
4. 可执行程序代码放在cmd/<程序名称>/*.cpp
5. 接口库、动态库、静态库代码放在lib/<库名称>/*.cpp
6. 不使用第三方依赖库
7. 界面库也要封装界面类，底层使用opengl渲染
8. 不使用make，使用ninja构建工程
