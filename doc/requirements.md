# Clockwork — 功能需求

本文档描述 Clockwork **产品范围与功能需求**。工程与接口细节见 [development.md](development.md)、[architecture.md](architecture.md)、[api.md](api.md)。

权威摘要：项目根目录 [README.md](../README.md)。

---

## 1. 项目概述

**Clockwork** 仿真平台仿真对象为**地球**及**地球上的实体**。

### 1.1 仿真实体类型（示例）

- 飞机、坦克、舰船
- 空面弹、空空弹、鱼类、水雷
- 箔条弹、角反射体等

### 1.2 仿真领域（示例）

- 可见光、红外光、电磁波
- 雷达、干扰、电子侦察等相关仿真

### 1.3 控制逻辑实现方式

- 使用 **Lua** 编写仿真实体的控制逻辑；或
- 使用**可视化蓝图脚本**控制平台逻辑。

**排期说明（与 [development-plan.md](development-plan.md) 一致）**：Lua 脚本**暂缓**实装；可视化蓝图**不暂缓**，按阶段 7 与里程碑推进。

## 2. 仿真平台组成

### 2.1 想定编辑

| 能力 | 说明 |
|------|------|
| 型号数据 | 根据已注册模型，配置模型参数生成型号数据 |
| 控制逻辑 | 编辑仿真实体的控制逻辑 |
| 航线与空域 | 预先设定航线、空域信息 |
| 通信网络 | 预先设置仿真通信网络 |
| 平台组装 | 将型号组装为仿真平台；可设置名称、二维图标、三维模型、属性等 |
| 场景部署 | 将仿真平台部署到场景中作为仿真实体；可设置部署位置、名称等 |

### 2.2 仿真引擎

- 读取**限定信息**初始化仿真实体。
- 接收控制指令：**初始化、开始、暂停、结束、设置仿真倍速**。
- **输出仿真态势信息**。
- 具备**分布式仿真**能力：可与其他计算机上的仿真引擎进行数据通信，使模型能够与**分布式的模型数据**交互。
- 具备**保存快照与恢复快照**能力（与 README 一致）。
- **快照与分布式互斥**：快照**仅**在**本地单机**仿真中可用；引擎处于**分布式联邦 / 对等互联**模式时，**不得**提供或接受快照保存与恢复（API 应拒绝或等价禁用）。

### 2.3 仿真模型（类型）

| 类型 | 职责 |
|------|------|
| **mover** | 仿真实体运动能力 |
| **sensor** | 探测其他实体 |
| **comdevice** | 实体间通信 |
| **processor** | 实体自身信息处理 |
| **weapon** | 微波武器或弹药发射架等 |
| **signature** | 可见光、红外、电磁等特性描述 |

### 2.4 态势显示（二维地图 + 三维地球）

- 显示二维地图、三维地球。
- 显示想定中的**航线**、**空域信息**。
- 显示**仿真实体**。
- 显示仿真实体**坐标轴**与**矢量信息**（速度矢量、转动矢量）。
- 显示**雷达波束**、**干扰波束**、**雷达包络**。

### 2.5 仿真运行程序

- 组合**仿真引擎**与**态势显示**：将引擎中的仿真实体绘制到态势显示中。
- 提供**人机交互界面**。
